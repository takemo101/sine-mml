## 追加機能

### MIDIストリーミング機能の追加
MMLのplayコマンドにMIDIストリーミング機能を追加したいです。
この機能により、MMLで指定されたメロディーをリアルタイムでMIDIデバイスに送信し、外部のMIDI対応機器やソフトウェアシンセサイザーで演奏できるようにします。
具体的な仕様は以下の通りです。

- `--midi-out`オプションを追加し、MIDIストリーミングを有効にします。
- MIDIデバイスの選択が可能で、複数のMIDI出力先から選べるようにします。
- MIDIメッセージの送信はリアルタイムで行い、MMLのテンポ設定に従って正確に演奏されるようにします。
- MIDIストリーミング中でも、他のオプション（音量調整、ループ再生など）が正常に機能するようにします。
- MIDIストリーミングの開始と停止はコマンドラインから制御可能にします。
- MIDIストリーミングのエラーハンドリングを実装し、、MIDIデバイスが接続されていない場合や通信エラーが発生した場合に適切なメッセージを表示します。
- MIDIストリーミングのパフォーマンスを最適化し、遅延を最小限に抑えるようにします。


### 連符（n連符）対応
MMLで連符（3連符、5連符など）を指定できるようにしたいです。

#### 採用構文: `{...}n` ブラケット方式

既存のループ構文 `[...]n` との一貫性を重視し、波括弧によるグループ化を採用します。

```mml
// 基本構文
{CDE}3           // 3連符: 3音を1拍（4分音符）に収める
{CDEF}4          // 4連符: 4音を1拍に収める
{CDEFG}5         // 5連符: 5音を1拍に収める

// デフォルト音長との連動
L8 {CDE}3        // 8分音符ベースの3連符（0.5拍に3音）

// ベース音長指定（オプション）
{CDE}3:2         // 2分音符（2拍）に3音を収める

// 休符を含む連符
{CDR}3           // 3連符の3音目が休符

// タイとの組み合わせ
{C4&8 D E}3      // 連符内でタイ記号を使用
```

#### 選定理由
- **一貫性**: ループ `[...]n` と同じ「括弧+数字=グループ操作」パターン
- **直感性**: 「この音をn連符として演奏」が視覚的に明確
- **拡張性**: ネストや将来的な機能追加に対応しやすい

#### 仕様
- 連符はCDEFGABやRなどの音符・休符と組み合わせて使用できます
- 連符は他の音符と組み合わせて使用でき、ループやタイ記号とも互換性を持たせます
- 連符の指定は、付点音符や休符とも組み合わせて使用できます
- 連符の演奏時に音量やテンポの調整が正しく反映されます
- 連符のエラーハンドリングを実装し、不正な指定があった場合に適切なメッセージを表示します
- ネスト深度は最大5階層（ループと同様）
